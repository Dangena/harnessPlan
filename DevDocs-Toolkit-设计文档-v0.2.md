## 一、项目概述

### 1.1 背景

公司软件开发的前期工作（产品需求、方案设计、投标文档、PPT、UI设计等）目前依赖员工手动基于旧模板编写，效率低、质量参差不齐。需要一个工具箱，让同事们能够按照公司材料模板，结合自身需求，快速完成各类文档的编写与输出。

### 1.2 边界定义

本工具箱聚焦于**软件开发前期中偏业务/产品侧**的文档工作，**不涉及技术设计**：

| ✅ 在范围内 | ❌ 不在范围内 |
|------------|-------------|
| 调研提纲、访谈纪要 | 技术概要设计（HLD） |
| 需求分析、需求清单 | 技术详细设计（LLD） |
| PRD | 架构设计文档 |
| UI原型 | 接口设计文档 |
| 项目立项方案 | 数据库设计文档 |
| 汇报材料 | 部署方案 |
| 测试用例 | |

> **说明**：技术概要设计、技术详细设计等属于开发阶段的技术文档，由技术团队负责，不在本工具箱的职责范围内。如未来有需要，可作为"开口扩展项"单独规划。

### 1.3 目标

- **效率提升≥50%**：标准化文档初稿自动生成，设计人员聚焦审核与决策
- **输出质量对齐资深水平**：模板+AI双重加持，确保格式规范、内容完整
- **开口可扩展**：预留方案类型扩展口，支持招投标、科研任务书、工作流设计等

### 1.4 目标用户与项目类型匹配

| 项目类型 | 典型特征 | 核心诉求 | Must工具 |
|---------|---------|---------|---------|
| **政务IT** | 政策合规强、流程规范、汇报对象多、文档格式严格 | 合规检查、格式规范、多版本输出 | ①调研、②需求、③PRD、⑤立项、⑥汇报 |
| **企业IT** | 业务导向、迭代快、成本敏感、灵活调整 | 效率优先、快速迭代、灵活裁剪 | ②需求、③PRD、④UI、⑦测试 |
| **科研** | 创新导向、里程碑明确、成果输出多样、周期长 | 技术路线、创新点、成果包装 | ①调研、⑤立项、⑥汇报 |
| **招投标** | 响应招标文件、评分标准明确、竞争性强 | 技术响应得分、文档专业度、差异化 | ②需求、③PRD、⑤立项 |
| **内部自建** | 需求明确、团队稳定、持续运维 | 文档完整、测试覆盖、知识沉淀 | ③PRD、⑤立项、⑦测试 |

### 1.5 使用模式

- **Skill形态（推荐80%场景）**：单一文档需求、快速验证、简单串联（≤3步）
- **Agent形态（推荐20%场景）**：完整流程自动化、并行多工具协同、长周期项目

## 二、整体架构

### 2.1 产出形式：Skill + Agent 混合形态

采用渐进式混合方案：

- **第一阶段 — Skill优先**：先做强大的Skill，支持所有文档类型的快速生成
- **第二阶段 — 按需扩展**：根据实际使用反馈，为高频协作场景开发轻量配套工具

### 2.2 三层架构

```
聚合层（7个工具Skill）  ← 用户直接调用这些
```

## 三、Skill vs Agent 形态决策

### 3.1 复杂度量化评分算法

```
复杂度得分 = 串联步骤数 × 1.0 + 并行工具数 × 1.5 + 决策分支数 × 2.0 + 上下文长度 × 0.5

- 得分 ≥ 8 → Agent形态
- 得分 < 8 → Skill形态
```

**评分示例：**
- 串联5步，并行2个，分支1个，上下文8个 → 5×1 + 2×1.5 + 1×2 + 8×0.5 = 14 → **Agent形态**
- 串联3步，并行1个，分支0个，上下文3个 → 3×1 + 1×1.5 + 0 + 3×0.5 = 6 → **Skill形态**

### 3.2 决策树

```
任务开始
│
├─ 是否单文档需求？
│ ├─ 是 → Skill形态 → 结束
│ └─ 否 ↓
│
├─ 是否跨会话/长周期？
│ ├─ 是 → Agent形态（持久化Agent）
│ └─ 否 ↓
│
├─ 计算复杂度得分
│ ├─ ≥ 8 → Agent形态
│ └─ < 8 → Skill形态
```

### 3.3 Agent形态内部实现策略

| 场景 | 推荐实现 | 说明 |
|------|---------|------|
| 完整流程自动化（7步串联） | Orchestration Agent | 小型专用Agent，负责流程编排 |
| 并行多工具协同 | 主Agent + Worker Agent ×N | 主Agent分发任务，Worker并行执行 |
| 复杂决策流 | 路由Agent + 状态机 | 基于决策表动态路由 |
| 长周期项目 | 持久化Agent + 知识库 | 定期快照上下文，支持恢复 |

### 3.4 状态机设计（Agent形态）

| 状态 | 说明 | 可转向前置状态 |
|------|------|---------------|
| IDLE | 初始状态，等待任务 | --- |
| RUNNING | 任务执行中 | IDLE |
| WAITING | 等待下游Agent结果/人工确认 | RUNNING |
| COMPLETED | 任务正常完成 | RUNNING |
| FAILED | 任务异常 | RUNNING |
| SUSPENDED | 暂停（等待外部输入/资源） | RUNNING, WAITING |

### 3.5 消息协议（Agent间通信）

采用JSON-RPC 2.0协议：

```json
// 任务下发
{
  "jsonrpc": "2.0",
  "id": "<unique_id>",
  "method": "agent.dispatch",
  "params": {
    "task_id": "task_001",
    "skill_name": "prd-flowchart-gen",
    "input": { /* 任务输入 */ },
    "priority": "normal",
    "timeout_ms": 120000
  }
}

// 结果返回
{
  "jsonrpc": "2.0",
  "id": "<unique_id>",
  "result": {
    "task_id": "task_001",
    "status": "completed",
    "output": { /* 任务输出 */ },
    "metadata": {
      "duration_ms": 45000,
      "skill_version": "1.0.0"
    }
  }
}
```

### 3.6 上下文管理（三层持久化）

| 层级 | 生命周期 | 存储位置 | 内容 |
|------|---------|---------|------|
| **会话层** | 单次任务执行 | 内存 | 当前任务输入/输出，中间状态 |
| **项目层** | 整个项目周期 | 文件/数据库 | 完整上下文快照（需求→PRD→立项→汇报） |
| **长期层** | 跨项目复用 | 知识库 | 模板、术语表、历史项目特征 |

**上下文快照机制：**
- 每个工具完成后自动生成快照：`{tool}_{timestamp}_context.json`
- 支持从任意历史快照恢复，跳过已完成步骤
- 项目层上下文定期序列化到磁盘（每30分钟或关键节点）

## 五、七大工具详解

### 5.1 工具①：调研AI辅助（tool-survey-assist）

**定位：** 需求入口，帮助设计人员快速完成用户业务调研

**核心能力：**
- 调研提纲生成（基于项目类型自动推荐访谈维度）
- 访谈记录实时摘要（录音转文字→AI提炼关键信息）
- 需求初稿生成（调研完成后自动输出需求原始清单）

**输出物：**
- 调研提纲（Word/Markdown）
- 访谈纪要（每场访谈一份）
- 需求原始清单

**Skill依赖：**
- `survey-outline-gen`
- `survey-interview-summary`
- `survey-requirements-draft`

### 5.2 工具②：需求分析（tool-demand-analysis）

**定位：** 将调研原始需求结构化、优先级化

**核心能力：**
- 需求清单生成（去重、归类、关联模块）
- 功能模块划分（按业务域/流程阶段分层）
- MoSCoW优先级排序（Must/Should/Could/Won't）
- 需求追踪矩阵（RTM）构建

**输出物：**
- 需求清单（带优先级）
- 功能模块划分图
- RTM表格（需求-用例追踪）

**Skill依赖：**
- `demand-deduplication`
- `demand-moscow-sort`
- `demand-rtm-builder`

### 5.3 工具③：PRD生成（tool-prd-generator）

**定位：** 核心交付物，生成完整PRD文档

**核心能力：**
- PRD文档自动生成（基于调研+需求清单）
- 附件自动生成：业务流程图、数据字典、功能列表表格、角色权限矩阵、页面清单

**PRD文档结构（标准版）：**

```
1. 文档概述
   1.1 项目背景
   1.2 产品目标
   1.3 术语与定义

2. 用户与市场
   2.1 目标用户
   2.2 用户画像
   2.3 用户旅程图

3. 功能需求
   3.1 功能模块总览
   3.2 [模块A] 详细需求
   3.3 [模块B] 详细需求
   ...

4. 非功能需求
   4.1 性能需求
   4.2 安全需求
   4.3 兼容性需求
   4.4 可用性需求

5. 附录
   5.1 流程图
   5.2 数据字典
   5.3 页面清单
   5.4 需求追踪矩阵
```

**输出物：** 完整PRD文档（飞书文档/Word）

**Skill依赖：**
- `prd-flowchart-gen`
- `prd-data-dict-gen`
- `prd-page-list-gen`
- `prd-role-matrix`

### 5.4 工具④：UI原型设计（tool-ui-prototype）

**定位：** 可视化沟通工具，快速输出线框图与交互流程

**核心能力：**
- 线框图生成（基于PRD页面清单）
- 交互流程图（页面跳转逻辑可视化）
- 低保真原型（可点击的简单原型，用于早期评审）
- Design Token输出（颜色/字体/间距规范）

**输出物：**
- 线框图（图片/figma链接）
- 交互流程图
- 低保真原型链接
- Design Token文档

**Skill依赖：**
- `ui-wireframe-gen`
- `ui-flowchart-gen`

### 5.5 工具⑤：项目立项方案（tool-project-setup）

**定位：** 决策审批文档，用于内部立项或对外汇报

**核心能力：**
- 项目立项书自动生成
- 包含章节：背景、目标、范围、计划、资源、风险、预算
- 甘特图生成（项目计划可视化）
- 工作量估算（基于需求清单自动估算人天）

**立项方案结构（标准版）：**

```
1. 项目概述
   1.1 项目背景
   1.2 建设目标
   1.3 建设范围（边界）
   1.4 预期收益

2. 需求分析
   2.1 业务需求
   2.2 用户需求
   2.3 功能需求概要

3. 项目计划
   3.1 实施周期
   3.2 里程碑计划
   3.3 甘特图
   3.4 资源需求

4. 技术方案（简版）
   4.1 总体架构
   4.2 技术选型
   4.3 安全设计

5. 项目组织
   5.1 组织架构
   5.2 人员分工

6. 风险与对策
   6.1 主要风险
   6.2 对策措施

7. 投资估算
   7.1 投资估算表
   7.2 资金来源
```

**开口扩展方向：**
- 招投标版本 → 增加技术响应、服务方案
- 科研版本 → 增加技术路线、创新点、考核指标

**Skill依赖：**
- `project-gantt-gen`
- `project-workload-estimate`

### 5.6 工具⑥：汇报材料生成（tool-report-material）

**定位：** 快速生成PPT和Word双版本汇报材料

**核心能力：**
- 自动读取立项方案/PRD关键章节
- 生成结构化PPT（20-30页，含封面、目录、内容、总结）
- Word版本同步生成（用于正式存档）
- 支持多种风格（商务政务风、科技风、简约风）

**输出物：**
- PPT文件（.pptx）
- Word文档（.docx）
- 飞书文档（可选）

**Skill依赖：**
- `report-ppt-gen`
- `report-word-gen`

### 5.7 工具⑦：测试用例生成（tool-test-case-gen）

**定位：** 质量保障前置，为开发团队提供完整测试用例

**核心能力：**
- 基于PRD功能清单，自动生成功能测试用例
- 正向用例 + 反向用例 + 边界值用例 + 异常场景用例
- 测试用例评审检查清单
- 缺陷等级分类建议

**输出物：**
- 测试用例表（Excel/飞书表格）
- 测试评审报告

**用例结构（标准格式）：**

## 六、工具串联与并行关系

### 6.1 标准链路（从调研到汇报）

```
调研AI辅助
    ↓
需求分析
    ↓
PRD生成 ←→ UI原型（并行）
    ↓
【双审】PRD + 原型评审
    ↓
项目立项方案
    ↓
汇报材料生成（PPT + Word）
    ↓
测试用例生成
```

### 6.2 精简链路（单文档需求）

```
调研AI辅助（可选）
    ↓
需求分析（可选）
    ↓
目标文档生成
```

### 6.3 串联触发条件

| 上游工具 | 下游工具 | 触发条件 |
|---------|---------|---------|
| ① 调研AI辅助 | ② 需求分析 | 访谈纪要≥3份，或需求初稿已生成 |
| ② 需求分析 | ③ PRD生成 | 需求清单已去重、优先级已排序 |
| ③ PRD生成 | ④ UI原型设计 | 页面清单≥5页 |
| ③ PRD生成 | ⑤ 项目立项方案 | PRD核心章节已完成（背景/目标/范围） |
| ⑤ 项目立项方案 | ⑥ 汇报材料生成 | 立项书已完成 |
| ③ PRD生成 | ⑦ 测试用例生成 | 功能模块≥3个 |

### 6.4 并行触发条件

| 并行工具组 | 并行条件 | 同步机制 |
|-----------|---------|---------|
| ③ PRD生成 ↔ ④ UI原型设计 | PRD页面清单已明确，且UI设计不影响PRD内容结构 | 独立并行，最终合并评审 |
| ⑥ 汇报材料生成 ↔ ⑤ 项目立项方案 | 项目背景/目标已明确，可同步生成 | 最终输出互补，汇报基于立项 |
| ⑦ 测试用例生成 ↔ ④ UI原型设计 | 功能模块已定义，原型同步用于用例验证 | 评审时交叉验证 |

### 6.5 共享Skill复用

| 共享Skill | 被调用工具 |
|----------|-----------|
| `core-flowchart-gen` | ③ PRD生成、⑤ 项目立项方案 |
| `core-table-builder` | ② 需求分析、③ PRD生成、⑦ 测试用例 |
| `core-markdown-gen` | ① 调研、② 需求、③ PRD、⑤ 立项 |
| `ui-flowchart-gen` | ③ PRD生成、④ UI原型 |

### 7.2 基础层（4个）

#### core-markdown-gen — Markdown文档生成

| 项目 | 说明 |
|------|------|
| 职责 | 将结构化内容渲染为格式规范的Markdown文档 |

| 输出 | 完整的.md文件 |
| 依赖 | 无 |
| 关键行为 | 自动处理标题层级、列表缩进、代码块、引用格式 |

#### core-table-builder — 表格生成

| 项目 | 说明 |
|------|------|
| 职责 | 根据数据生成Markdown/GitHub风格表格 |

| 关键行为 | 支持合并单元格提示、自动对齐、空值处理 |

#### core-flowchart-gen — 流程图生成

| 项目 | 说明 |
|------|------|
| 职责 | 根据流程描述生成Mermaid流程图代码 |
| 输入 | 流程步骤列表、分支条件、起止节点 |
| 输出 | Mermaid代码块（可渲染为流程图） |

#### core-toc-generator — 目录生成

| 项目 | 说明 |
|------|------|

### 7.3 领域层（4个）

#### prd-data-dict-gen — 数据字典生成

| 项目 | 说明 |

| 项目 | 说明 |

### 7.4 聚合层（1个）

#### tool-prd-generator — PRD生成工具

| 项目 | 说明 |
|------|------|
| 职责 | 编排所有Skill，端到端生成完整PRD |
| 输入 | 用户自然语言描述（项目名/目标/需求/约束等） |
| 输出 | PRD文档.md + 业务流程图.md + 数据字典.md + 角色权限矩阵.md |
| 依赖 | 以上8个Skill全部 |
| 关键行为 | 见下方编排流程 |

**编排流程：**

```
1. 接收用户输入 → 解析项目信息
2. 信息补全 → 主动询问缺失项（非功能需求、约束、角色等）
3. 生成PRD框架 → 调用 core-toc-generator + core-markdown-gen
4. 生成功能清单 → 调用 core-table-builder
5. 生成业务流程 → 调用 prd-flowchart-gen
6. 生成页面清单 → 调用 prd-page-list-gen
7. 生成数据字典 → 调用 prd-data-dict-gen
8. 生成权限矩阵 → 调用 prd-role-matrix
9. 组装完整PRD → 调用 core-markdown-gen
10. 输出所有文件
```

## 九、设计决策记录

| 决策 | 选项 | 结论 | 理由 |
|------|------|------|------|
| 产出形式 | Skill / Web应用 / 混合 | 混合形态 | Skill快速上手 + 后期扩展协作能力 |
| 启动策略 | 自底向上 / 自顶向下 / 垂直切片 | 垂直切片 | 最快验证核心价值，投入可控 |
| 首期文档类型 | PRD / 方案设计 / 投标文档 / PPT / UI | PRD | 核心交付物，结构最完整，使用频率最高 |
| 模板策略 | 固定模板 / 灵活模板 / 两者兼顾 | 两者兼顾 | 公司有标准模板但不同场景需要灵活调整 |
| 输出格式 | 仅Markdown / 按用户要求 | 按用户要求 | 不同场景需要不同格式（md/docx/pptx/xlsx） |
| 形态选择 | 纯Skill / 纯Agent / 混合 | 80% Skill + 20% Agent | 简单场景直接调用，复杂场景自动化 |
| Agent实现 | 外部框架 / 自构建 | 自构建轻量Agent | 场景聚焦，与Skill库天然对齐，调试成本低 |
